Import Analysis (Operations)

Upload a .gsd analysis file, or a CSV of operations (Code, Title, SMV). You'll get a read-only preview of every operation code found inside before anything is imported — nothing in your library is changed until you confirm. Every operation code not already in your library gets auto-created here in one shared "Import Operation" folder, keeping the original code visible on the operation (no elemental breakdown is available from an SMV-only source, so it's stored as one "imported" line carrying the real code). Codes that already exist are simply left as-is (linked, never overwritten).

CSV format (if not using a .gsd export): Code,Title,SMV OS-ADULTS-KTOPS-ASM0001,JOIN 1ST SHOULDER JOIN BY F/S,0.2543 OS-ADULTS-KTOPS-CO0021,NECK BINDING-F/L,0.4333
Codes found inside this file — read only

Import Analysis (CSV)

Upload a .gsd analysis file exported from GSD Cost, or a CSV with a Style header row followed by the operation lines. You'll get a read-only preview of every operation code inside the file before anything is imported — nothing in your library changes until you confirm. Style import never creates or edits Operations. A code that already exists in your library gets linked as-is; a code that doesn't exist yet is kept on the Style as a plain code reference so you can still see what's inside — it just isn't turned into a library Operation. The Style itself is saved under an "Import Style" folder so imported work stays separate from the rest of your library.

CSV format (if not using a .gsd export): StyleNo,Description,Buyer TOP,CREW NECK T-SHIRT,ZARA Ln,Type,Code,Title,Freq,SMV 1,OP,OS-ADULTS-KTOPS-ASM0001,JOIN 1ST SHOULDER JOIN BY F/S,1,0.2543 2,OP,OS-ADULTS-KTOPS-CO0021,NECK BINDING-F/L,1,0.4333
Styles & operation codes found inside this file — read only
